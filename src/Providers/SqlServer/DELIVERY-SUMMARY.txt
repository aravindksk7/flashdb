================================================================================
KEY FEATURES
================================================================================

GOLDEN IMAGE CREATION:
  ✓ Method 1 (BackupRestore): Restores backup file to VHDX
    - Reliable, works with offline backups
    - Performance: 30-60 minutes for 10GB databases

  ✓ Method 2 (ReplicaBackup): BACKUP FROM MIRROR from read-only replica
    - Fast alternative for large databases
    - Includes replica lag detection
    - Performance: 15-40 minutes for 10GB databases

  ✓ Method 3 (TableByTableCopy): Direct table enumeration and copy
    - Most flexible, works with read-only accounts
    - No admin rights required on source
    - Performance: 60-180 minutes for 10GB databases

DATABASE MANAGEMENT:
  ✓ Attach database from VHDX differencing disk
    - CREATE DATABASE ... FOR ATTACH T-SQL
    - Automatic MDF/LDF file discovery
    - Handles multiple data files

  ✓ Detach database from SQL instance
    - sp_detach_db T-SQL command
    - Automatic connection cleanup
    - Graceful session termination with logging

  ✓ Backup/Restore operations
    - BACKUP DATABASE with compression
    - RESTORE DATABASE with multi-user mode management

VERIFICATION & MONITORING:

INTEGRATION WITH FLASHDB CORE
================================================================================

Provider Registration:
  The SqlServerProvider integrates with FlashDB core via the
  ICloneProvider interface (implicit via method signatures)

Required Core Integration Points:
  1. Golden Image Management
     - CreateGoldenImage() supports three methods
     - Returns metadata for persistence

  2. Clone Provisioning
     - AttachDatabase() provisions clone
     - Returns database connection information

  3. Clone Rollback
     - DetachDatabase() before VHDX snapshot revert
     - Handles connection cleanup

  4. Metadata Tracking
     - GetDatabaseInfo() returns size, table count, row count hash
     - ComputeRowCountHash() verifies consistency

  5. Verification
     - ValidateConnection() tests connectivity
     - GetReplicaLag() monitors synchronization health

Typical Workflow:
  1. New-FlashdbGoldenImage() → CreateGoldenImage()
  2. New-FlashdbClone() → AttachDatabase()
  3. New-FlashdbCheckpoint() → [VHDX snapshot via core]
  4. Restore-FlashdbCheckpoint() → DetachDatabase() + revert + AttachDatabase()
  5. Remove-FlashdbClone() → DetachDatabase()

SUPPORT & TROUBLESHOOTING
================================================================================

Common Issues & Solutions:

Issue: "Cannot open database connection"
  → Check connection string format
  → Verify server name and instance
  → Test with sqlcmd or SSMS first

Issue: "Backup file not found"
  → Verify backup path exists
  → Check file permissions
  → Verify backup file is readable

Issue: "VHDX path not found"
  → Ensure VHDX is mounted
  → Verify path is correct and accessible
  → Check file permissions

Issue: "No MDF files found on VHDX"
  → Verify VHDX contains database files
  → Check VHDX is mounted to correct drive letter
  → Ensure file permissions allow read access

Issue: "Replica lag too high"
  → Wait for replica synchronization
  → Check network connectivity
  → Monitor sys.dm_database_mirroring_status

Issue: "Active connections prevent detach"
  → Check for long-running queries
  → Increase session termination timeout
  → Review SQL Server activity monitor

For Detailed Troubleshooting:
  → Enable $VerbosePreference = 'Continue'
  → Review logs in provider output
  → Check SQL Server error logs
  → Verify permissions via SSMS
