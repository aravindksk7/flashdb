================================================================================
KEY DESIGN DECISIONS
================================================================================

1. HYBRID EXECUTION MODEL
   - Synchronous (direct): Quick validation, dry-run planning (< 5 sec)
   - Asynchronous (queued): Full validation, repair execution (5-300 sec)
   - Query param: queue=true|false controls execution mode

2. LOCK MANAGEMENT (3-Level Hierarchy)
   - clone:${cloneId} - General clone lock (prevents concurrent modifications)
   - clone-validation:${cloneId} - Prevents concurrent validations
   - clone-repair:${cloneId} - Prevents concurrent repairs
   - Conflicts return 409 with actionable lock info

3. COMPREHENSIVE AUDIT TRAIL
   Operations tracked:
   - validation-start, validation-complete, validation-error
   - repair-plan, repair-execute, repair-complete, repair-cancel

   Metrics captured:
   - Finding counts (error, warning, info)
   - Action counts and success rates
   - Duration, timestamps, operator ID
   - Trigger type (manual, scheduled, automatic)

4. ERROR HANDLING (7 Standardized Codes)
   - E001_CLONE_NOT_FOUND (404)
   - E002_VALIDATION_IN_PROGRESS (409)
   - E003_REPAIR_IN_PROGRESS (409)
   - E004_INVALID_REPAIR_STATE (422)
   - E005_VALIDATION_REQUIRED (422)
   - E006_CLONE_LOCKED (409)
   - E007_SERVICE_ERROR (500)
   All errors include actionable details and suggestions

5. TYPE-SAFE CONTRACTS
   Full TypeScript interfaces for:
   - Request/response shapes
   - Validation findings with severity levels
   - Repair plans with actions and risk levels
   - Audit records with operation-specific metrics
   - Error responses with standardized codes

ARCHITECTURE HIGHLIGHTS
================================================================================

✓ Separation of Concerns
  Service logic (cloneValidationService) → API routes → GUI components

✓ Async-Capable
  Direct execution for quick operations, task queue for long-running tasks

✓ Concurrent-Safe
  3-level lock hierarchy prevents race conditions

✓ Observable
  Comprehensive audit trail with operation-specific metrics

✓ Resilient
  Error handling with actionable suggestions for all failure modes

✓ Type-Safe
  Full TypeScript contracts throughout frontend and backend

✓ Scalable
  Bulk operation support for validating multiple clones

✓ User-Friendly
  Progress tracking, actionable error messages, visual indicators
